使い方

新規テンプレート作成の注意点 主催者マスタ での「GSS原本ID」は、生成する出品商品の詳細シート以外の、
補助的データや表紙などを入力してください。

出品商品の詳細シートは、 商品データ原本 へ追加し、
追加したシートIDを主催者マスタの「出品表シート原本ID」へ入力してください。
項目 Item Type :製品・ダイヤで処理を分けている。ものばんくなどは「その他」で。
SEQ一括入力 そのまんま。一括して商品を登録。ソートも入力順で処理される。
メモUP:「Comment(番号)」を商品へ反映
製品→コメント（comment）
ダイヤ→メモ（description）
その他（ものばんく,JWA,JBA）はコメント
ドライブUP
初期、ドライブへGSS作成し、URL保存
次回以降はそのURLへ上書き保存
初期作成時はGAS(LoungeAPI)へPOST送信（この時点ではGSS原本IDからコピーするだけ）
GASの編集画面
実際の商品データの書き込み「drive_gss_write.php」

Item Typeで分けていたもの
▼出品商品データの取得▼
出品表IDから取得。製品・その他：ソート順。ダイヤ：親石降順

▼出品表の商品シートの原本GSSのシートID▼
原本GSS から商品データのシートのテンプレートとして使用するシートID
※[その他]は主催者マスタのidごとに設定する。

▼商品データの生成▼
金性・大カテ・中カテ以外は製品・ダイヤで分岐
※[その他]はシートID同様、主催者マスタのidごとに設定する。
▼書込みの範囲▼
書き込みGSSごとに異なる。つまり、[その他]は主催者マスタのidごと。

▼出品表上部書込み▼
項目の有無、位置で完全にバラバラ。細かく調整
